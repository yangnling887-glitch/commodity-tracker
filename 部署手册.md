# 部署手册：大宗商品追踪网站

## 你需要准备的
- 一个 **GitHub 账号**（免费注册：https://github.com）
- 这个 `commodity-tracker` 文件夹（你现在看到的）

---

## 第一步：注册并登录 GitHub

访问 https://github.com，注册账号并登录。

---

## 第二步：创建仓库

1. 登录后点击右上角 **"+"** 按钮 → **"New repository"**
2. 填写：
   - **Repository name**：`commodity-tracker`
   - **Description**：大宗商品行情追踪（可留空）
   - 选择 **Public**（公开，GitHub Pages 免费版需要公开）
   - **不要**勾选 "Add a README file"
3. 点击 **"Create repository"**

---

## 第三步：上传文件

创建仓库后，页面会显示空仓库。点击 **"uploading an existing file"** 链接。

把 `commodity-tracker` 文件夹里的**所有内容**拖进上传区域：
- `.github/` 文件夹
- `config/` 文件夹
- `data/` 文件夹
- `scripts/` 文件夹
- `web/` 文件夹

等待上传完成，在页面底部点击 **"Commit changes"**。

---

## 第四步：开启 GitHub Pages

1. 进入仓库，点击顶部 **"Settings"** 标签
2. 左侧菜单找到 **"Pages"**
3. **Source** 选择：**"Deploy from a branch"**
4. **Branch** 选择：`main`，文件夹选择 `/ (root)`

   > ⚠️ 注意：需要把 web 目录内容移到根目录，或者改成 `/web` 目录
   > 实际操作：Branch 选 `main`，目录选 `/web`（如果有此选项）
   > 如果没有 `/web` 选项，见下方"备选方案"

5. 点击 **"Save"**
6. 等待约 2 分钟，页面会显示你的网址：
   `https://你的用户名.github.io/commodity-tracker/`

---

## 第五步：手动触发第一次数据更新

1. 进入仓库，点击顶部 **"Actions"** 标签
2. 左侧找到 **"每日更新大宗商品数据"**
3. 点击右边的 **"Run workflow"** → **"Run workflow"**
4. 等待约 3-5 分钟，绿色 ✅ 表示成功
5. 刷新你的网站，就能看到真实数据了

---

## 备选方案：如果 Pages 只能选根目录

把 `web/` 目录里的三个文件（`index.html`、`style.css`、`app.js`）移到仓库根目录，
同时把 `app.js` 第一行的路径改为：
```
const DATA_URL = "data/commodities.json";
```

---

## 后续操作

### 添加新品种
编辑 `config/commodities.yaml`，按现有格式添加一条记录，提交后下次自动更新时生效。

### 数据更新频率
默认每天北京时间 18:30（周一到周五）自动更新。
修改 `.github/workflows/update_data.yml` 中的 `cron` 表达式可调整频率。

### 查看运行日志
在 **Actions** 标签页可以看到每次运行的详细日志，方便排查数据抓取问题。

---

## 常见问题

**Q: Actions 运行失败怎么办？**
A: 点进去查看日志，通常是某个品种的数据接口临时不可用。数据会保留上次成功的结果。

**Q: 某个品种一直没有数据怎么办？**
A: 稀土、钨的公开数据源稳定性较差，可以联系我调整抓取方式。

**Q: 能用手机访问吗？**
A: 可以，网站已做响应式适配。
